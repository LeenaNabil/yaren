# تفاصيل الإتاحة في مشروع يارين تورز

## نظرة عامة
هذا الملف يشرح التعديلات والإضافات الخاصة بالإتاحة في الموقع مع التركيز على:
- شريط الإتاحة العائم
- صفحة تصريح الإتاحة
- خيارات التنقل بالكيبورد
- القيم العالمية والإعدادات في `globals.css`
- دمج عناصر الإتاحة في التخطيط العام

## الملفات الرئيسية

### 1. `components/ui/accessibility-toolbar.tsx`
شريط الإتاحة العائم هو مكون React يعمل على جانب الصفحة ويقدم أدوات الإتاحة التالية:
- تكبير وتصغير حجم الخط بين 12px و24px
- تفعيل/تعطيل التباين العالي
- تفعيل/تعطيل وضع الأبيض والأسود (Grayscale)
- إعادة ضبط الإعدادات

#### الميزات التقنية:
- يستخدم `useState` لحفظ الحالة المحلية
- يستخدم `useEffect` لتحميل الإعدادات المحفوظة من `localStorage`
- يغير حجم الخط عبر `document.documentElement.style.fontSize`
- يضيف أو يزيل فصول CSS (`high-contrast`, `grayscale`) على `document.documentElement`
- يحتفظ بالإعدادات في `localStorage` حتى بعد إعادة تحميل الصفحة

#### عناصر الواجهة:
- زر ثابت في الأسفل يمين الشاشة
- نافذة حوار توضح حالة الإتاحة وتسمح بالتفاعل
- دعم `aria-label`, `aria-pressed`, `role="dialog"`
- قسم للمحتوى القابل للقراءة بواسطة قارئ الشاشة (`aria-live`)

### 2. `app/layout.tsx`
التخطيط الرئيسي للموقع يضيف العناصر التالية:
- رابط تخطي إلى المحتوى الرئيسي `skip-link` لتحسين التنقل بالكيبورد
- تضمين `AccessibilityToolbar` في كل صفحات الموقع
- دعم RTL عبر `dir="rtl"`

#### التفاصيل:
- `html` يحتوي على `lang="ar"` و `dir="rtl"`
- `body` يضم `skip-link` قبل المحتوى الرئيسي
- يتم تحميل شريط الإتاحة في جميع الصفحات عبر `RootLayout`

### 3. `app/accessibility/page.tsx`
صفحة تصريح الإتاحة تم تصميمها لتكون واضحة ومباشرة وتحتوي على الأقسام التالية:
- عنوان رئيسي واضح مع أيقونة Eye
- التزام بالإتاحة وفقا للمعايير الإسرائيلية `תקן ישראלי 5568`
- قائمة الأدوات والتقنيات المستخدمة
- ميزات الإتاحة المتاحة
- معلومات الاتصال لقسم الإتاحة
- آخر تحديث وتفاصيل الامتثال القانوني

#### تحسينات التصميم:
- تم رفع الحشو العلوي (`pt-32`) لتجنب التصاق المحتوى بالنافبار
- استخدام تنسيق بطاقات بيضاء مع ظل خفيف وحدود خفيفة
- تقسيم المحتوى إلى أعمدة متجاوبة مع `grid`
- زيادة حجم النصوص والعناوين لجعل القراءة أسهل
- إضافة بطاقات جانبية أكثر وضوحا في الشريط الجانبي

### 4. `app/globals.css`
الملف يحتوي على إعدادات CSS عامة ودعم الإتاحة:
- تعريف ألوان وقيم ثيمية باستخدام متغيرات `--navy`, `--orange` وغيرها
- دعم RTL مع `text-align: right` للعناصر داخل `[dir="rtl"]`
- أنماط التمرير المخصصة
- تأثيرات تحريك خاصة بالموقع
- أنماط إضافية لبطاقات وزرار
- أنماط خاصة بالإتاحة:
  - `.high-contrast` لتفعيل تباين عالي
  - `.grayscale` لتحويل الصفحة إلى الأبيض والأسود
  - `*:focus` لإضافة حدود واضحة عند التنقل بالكيبورد

## خصائص الإتاحة الرئيسية

### شريط الإتاحة
- زر ثابت لفتح لوحة الإتاحة
- تكبير وتصغير الخط
- تباين عالي
- تحويل إلى أبيض وأسود
- حفظ الإعدادات محليا وإعادة تعيينها

### التنقل بالكيبورد
- رابط تخطي إلى المحتوى الرئيسي في `app/layout.tsx`
- فوكس واضح للعناصر عبر CSS
- لوحة الإتاحة تدعم الأزرار و `aria-pressed`

### محتوى صفحة تصريح الإتاحة
- عناوين هرمية واضحة `h1`, `h2`, `h3`
- نصوص وصفية واضحة
- بطاقة معلومات الاتصال التي تساعد المستخدمين على إرسال استفسارات
- تاريخ آخر تحديث يظهر بصيغة عربية

### التوافق القانوني
- ذكر `תקן ישראלי 5568`
- توافق مع `WCAG 2.1 AA`
- تضمين معلومات عن قانون المساواة في الحقوق

## استخدام الملف
يمكنك استخدام هذا الملف للرجوع إلى:
- ما تم تنفيذه في الموقع
- كيفية عمل شريط الإتاحة
- أين تم تضمين رابط تخطي المحتوى
- إعدادات CSS والتنسيق العام

## ملاحظات إضافية
- في حال إضافة ميزات إضافية لاحقا يمكن توسيع `AccessibilityToolbar` لدعم:
  - وضع القارئ الصوتي
  - زيادة التباعد بين السطور
  - تغيير لوحة الألوان بالكامل
  - إيقاف الحركات التلقائية
- يمكن أيضا إضافة صفحة `AccessibilityPolicy` مستقلة لتفصيل سياسات الإتاحة بالكامل.
